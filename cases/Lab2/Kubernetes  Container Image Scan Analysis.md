## 3. รายละเอียดช่องโหว่สำคัญ (Key Vulnerabilities Breakdown)

### 3.1 รายงานฝั่ง `af` (ตัวอย่างช่องโหว่ระดับระบบปฏิบัติการ Ubuntu)
* **CVE-2026-27456** (`util-linux` / `bsdutils` / `libblkid1`)
  * **รายละเอียด:** ปัญหา Time-of-Check-Time-of-Use (TOCTOU) ใน `/usr/bin/mount` ขณะตั้งค่า loop devices ทำให้ผู้ใช้ทั่วไปสามารถใช้สิทธิ์ root อ่านไฟล์หรือ Block devices ที่ได้รับการปกป้องได้
* **CVE-2026-3184** (`util-linux` / `login`)
  * **รายละเอียด:** ปัญหาการบายพาส Access Control ผ่าน PAM เนื่องจากการจัดการชื่อ Hostname ไม่ถูกต้องในคำสั่ง `login`
* **CVE-2026-4046** (`glibc` / `libc-bin`)
  * **รายละเอียด:** เกิดภาวะ Denial of Service (DoS) แอปพลิเคชันค้างหรือล่ม เมื่อมีการแปลงอักขระชุด IBM1390 หรือ IBM1399 ผ่านฟังก์ชัน `iconv()`

### 3.2 รายงานฝั่ง `bf` (ตัวอย่างช่องโหว่ระดับ CRITICAL พร้อมดัชนีความเสี่ยง)
1. **CVE-2021-44228 (Log4Shell)**
   * **ระดับความรุนแรง:** CRITICAL (CVSS: 10.0)
   * **EPSS Score:** 0.99999 (มีโอกาสถูกโจมตีจริงสูงมาก)
   * **สถานะ CISA KEV:** อยู่ในรายการ Known Exploited Vulnerabilities (ถูกใช้โจมตีจริงแล้ว)
   * **ผลกระทบ:** สั่งรันโค้ดอันตรายระยะไกล (Remote Code Execution - RCE) ใน Log4j 2.x
2. **CVE-2021-45046 (Log4j DoS)**
   * **ระดับความรุนแรง:** CRITICAL (CVSS: 9.0)
   * **EPSS Score:** 0.99977
   * **สถานะ CISA KEV:** อยู่ในรายการ CISA KEV
   * **ผลกระทบ:** เกิดการปฏิเสธการให้บริการ (DoS) ใน Log4j 2.x ซึ่งเป็นผลมาจากการแก้ไขแพตช์ Log4Shell ที่ยังไม่สมบูรณ์
3. **CVE-2020-15999 (`libfreetype6`)**
   * **ระดับความรุนแรง:** CRITICAL (CVSS: 9.6)
   * **EPSS Score:** 0.5063
   * **สถานะ CISA KEV:** อยู่ในรายการ CISA KEV
   * **ผลกระทบ:** เกิด Heap-based buffer overflow ในฟังก์ชัน `Load_SBit_Png` ขณะประมวลผลไฟล์รูปภาพ PNG
4. **CVE-2017-5645 (`log4j-core`)**
   * **ระดับความรุนแรง:** CRITICAL (CVSS: 9.8)
   * **EPSS Score:** 0.8904
   * **ผลกระทบ:** ช่องโหว่ Deserialization บน Socket receiver ของ Log4j

## 4. ข้อสรุปและการประเมินความเสี่ยง (Summary & Risk Assessment)

1. **ความเสี่ยงระดับวิกฤต (High Critical Risk):** 
   แม้ชื่อ Container Image จะตั้งว่า `cve-2017-5645-patch-log4j:latest` (ซึ่งตั้งใจจะสื่อว่าแพตช์ CVE-2017-5645 แล้ว) แต่ผลจากการสแกนพบว่า Image นี้ยังมีช่องโหว่ร้ายแรงระดับโลกรวมถึง **Log4Shell (`CVE-2021-44228`)** และ **`CVE-2021-45046`** ซึ่งเปิดโอกาสให้ผู้โจมตีทำการควบคุมระบบแบบ RCE ได้ทันที

2. **ข้อเสนอแนะในการแก้ไข (Actionable Recommendations):**
   * **อัปเดต Java Libraries:** ทำการอัปเดตแพ็กเกจ `org.apache.logging.log4j:log4j-core` ให้เป็นเวอร์ชันล่าสุดที่ปลอดภัย (เวอร์ชัน `>= 2.17.1` หรือสูงกว่า)
   * **อัปเดต OS Packages:** อัปเดตแพ็กเกจระดับระบบปฏิบัติการใน Image เช่น `util-linux`, `glibc`, `libfreetype6` และแพ็กเกจอื่น ๆ ให้เป็นเวอร์ชันล่าสุด
   * **ปรับปรุงกระบวนการ Build Image:** นำการสแกนด้วย Trivy/CVE Scanner เข้าไปใส่ใน CI/CD Pipeline และกำหนด Policy ห้ามผ่านการ Build หากพบช่องโหว่ระดับ Critical/High
